Dynamic programming (DP) is an algorithmic technique that solves complex problems by breaking them into overlapping subproblems, solving each subproblem once, and storing the results for reuse. It is applicable when a problem exhibits **optimal substructure** (an optimal solution is composed of optimal solutions to subproblems) and **overlapping subproblems** (the same subproblems are solved multiple times).

## Memoization vs. tabulation

There are two equivalent ways to implement dynamic programming:

**Memoization (top-down)** starts from the original problem and recurses into subproblems, caching results so each subproblem is only computed once.

```go
// Top-down: solve recursively, cache results
var memo = map[int]int{}
func fib(n int) int {
    if n <= 1 { return n }
    if v, ok := memo[n]; ok { return v }
    memo[n] = fib(n-1) + fib(n-2)
    return memo[n]
}
```

**Tabulation (bottom-up)** builds a table of subproblem solutions iteratively, starting from the smallest subproblems and composing them upward.

```go
// Bottom-up: fill table from base cases
func fib(n int) int {
    dp := make([]int, n+1)
    dp[0], dp[1] = 0, 1
    for i := 2; i <= n; i++ {
        dp[i] = dp[i-1] + dp[i-2]
    }
    return dp[n]
}
```

<Note>
Most algorithms in `github.com/TheAlgorithms/Go/dynamic` use the tabulation approach, offering predictable time and space complexity without recursion stack overhead.
</Note>

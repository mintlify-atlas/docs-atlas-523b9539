## Prerequisites

You need Go 1.19 or later. The repository uses generics, which were introduced in Go 1.18, and the `go.mod` declares `go 1.19`.

<Tabs>
  <Tab title="macOS">
    ```bash
    brew install go
    go version  # should print go1.19 or later
    ```
  </Tab>
  <Tab title="Linux (Debian/Ubuntu)">
    ```bash
    sudo apt-get install golang
    go version
    ```
  </Tab>
  <Tab title="Windows">
    ```bash
    choco install golang
    go version
    ```
  </Tab>
  <Tab title="Manual">
    Download the installer from [go.dev/dl](https://go.dev/dl/) and follow the instructions for your platform.
  </Tab>
</Tabs>

## Set up the repository

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/TheAlgorithms/Go.git
    cd Go
    ```
  </Step>
  <Step title="Run all tests">
    This verifies every algorithm compiles and passes its test cases. The repository has no external dependencies, so no `go mod download` step is needed.

    ```bash
    go test ./...
    ```

    A successful run prints one `ok` line per package:

    ```
    ok      github.com/TheAlgorithms/Go/cache         0.002s
    ok      github.com/TheAlgorithms/Go/checksum      0.001s
    ok      github.com/TheAlgorithms/Go/cipher        0.003s
    ok      github.com/TheAlgorithms/Go/dynamic       0.008s
    ok      github.com/TheAlgorithms/Go/graph         0.012s
    ok      github.com/TheAlgorithms/Go/search        0.002s
    ok      github.com/TheAlgorithms/Go/sort          0.009s
    ...
    ```
  </Step>
  <Step title="Run the benchmarks">
    Benchmarks are included alongside the tests. Run them with the `-bench` flag:

    ```bash
    go test -bench=. ./...
    ```

    To benchmark a specific package — for example, to compare sorting algorithms — target just that package:

    ```bash
    go test -bench=. github.com/TheAlgorithms/Go/sort
    ```

    Sample output:

    ```
    BenchmarkBubble-8          136082      8802 ns/op
    BenchmarkMerge-8           613354      1923 ns/op
    BenchmarkQuick-8           686408      1743 ns/op
    BenchmarkHeap-8            487230      2461 ns/op
    BenchmarkTimsort-8         924075      1296 ns/op
    ```

    <Tip>
      Add `-benchmem` to see memory allocations per operation: `go test -bench=. -benchmem ./sort`
    </Tip>
  </Step>
  <Step title="Explore an algorithm">
    You can call any package directly from your own Go code. The module path is `github.com/TheAlgorithms/Go`.

    Create a file `main.go` anywhere outside the repository:

    ```go
    package main

    import (
        "fmt"
        "github.com/TheAlgorithms/Go/sort"
    )

    func main() {
        numbers := []int{64, 34, 25, 12, 22, 11, 90}
        sorted := sort.Bubble(numbers)
        fmt.Println(sorted) // [11 12 22 25 34 64 90]
    }
    ```

    Then initialize a module and run it:

    ```bash
    go mod init example
    go mod edit -replace github.com/TheAlgorithms/Go=./Go
    go run main.go
    ```

    <Note>
      The `-replace` directive points your local module at the cloned repository directory. In a real project you would use a tagged release instead.
    </Note>
  </Step>
</Steps>

## Working example: Bubble sort

The `sort` package exports generic sorting functions. Here is a self-contained example showing Bubble sort on an integer slice, based on the actual source in `sort/bubblesort.go`:

```go
package main

import (
    "fmt"
    "github.com/TheAlgorithms/Go/sort"
)

func main() {
    // sort.Bubble accepts any constraints.Ordered type: integers, floats, or strings.
    ints := []int{5, 3, 8, 1, 9, 2, 7, 4, 6}
    fmt.Println("Before:", ints)

    sorted := sort.Bubble(ints)
    fmt.Println("After: ", sorted)
    // Output:
    // Before: [5 3 8 1 9 2 7 4 6]
    // After:  [1 2 3 4 5 6 7 8 9]

    // Works on strings too, because string satisfies constraints.Ordered.
    words := []string{"banana", "apple", "cherry"}
    fmt.Println(sort.Bubble(words))
    // Output: [apple banana cherry]
}
```

The function signature in `sort/bubblesort.go` is:

```go
// Bubble is a simple generic definition of Bubble sort algorithm.
func Bubble[T constraints.Ordered](arr []T) []T
```

The `constraints.Ordered` constraint covers all integer types, all floating-point types, and `string`. It is defined in the internal `constraints` package:

```go
// Ordered is a generic type constraint for all ordered data types in Go.
type Ordered interface {
    Integer | ~string | Float
}
```

## Running a single test

To run a specific test function, use the `-run` flag with a pattern:

```bash
# Run only the Bubble sort test
go test -run TestBubble github.com/TheAlgorithms/Go/sort

# Run all tests whose names contain "Merge"
go test -run Merge github.com/TheAlgorithms/Go/sort
```

## Formatting the code

All code must be formatted with `gofmt`. This is enforced by CI.

```bash
# Format a single file in place
gofmt -w sort/bubblesort.go

# Format an entire package
go fmt github.com/TheAlgorithms/Go/sort
```

**TheAlgorithms/Go** is an open-source repository of classic algorithms and data structures written in Go. Every implementation is designed to be readable, well-documented, and covered by tests. The goal is education: each file explains the algorithm it implements, not just the mechanics of the code.

The module path is `github.com/TheAlgorithms/Go`. The repository requires no external dependencies — every package uses only the Go standard library (plus the internal `constraints` package for generics).

<Note>
  These implementations are for learning and reference. For production use, prefer the algorithms in Go's standard library (e.g., `sort.Slice`, `container/heap`).
</Note>

## Who it's for

- Students learning algorithms and data structures
- Developers preparing for technical interviews
- Go programmers exploring idiomatic patterns such as generics and interfaces
- Contributors who want to practice implementing algorithms in a real codebase

## Repository layout

The repository is organized by category. Each directory is a separate Go package.

```
github.com/TheAlgorithms/Go/
├── sort/          # Sorting algorithms (25+ implementations)
├── search/        # Searching algorithms
├── graph/         # Graph traversal and shortest-path algorithms
├── dynamic/       # Dynamic programming problems
├── structure/     # Data structures
├── math/          # Mathematical algorithms and number theory
├── cipher/        # Classical encryption ciphers
├── strings/       # String algorithms and pattern matching
├── compression/   # Data compression (Huffman coding)
├── hashing/       # Hashing algorithms (SHA-256)
├── conversion/    # Data format conversions
├── cache/         # Cache replacement policies (LRU, LFU)
├── checksum/      # Checksum algorithms (CRC8, Luhn)
├── constraints/   # Generic type constraints
└── other/         # Miscellaneous algorithms
```

## Design principles

**Generics.** Many algorithms use Go generics via the internal `constraints` package. For example, `sort.Bubble` accepts any `constraints.Ordered` type — integers, floats, or strings — without code duplication.

```go
// Bubble is a simple generic definition of Bubble sort algorithm.
func Bubble[T constraints.Ordered](arr []T) []T {
    swapped := true
    for swapped {
        swapped = false
        for i := 0; i < len(arr)-1; i++ {
            if arr[i+1] < arr[i] {
                arr[i+1], arr[i] = arr[i], arr[i+1]
                swapped = true
            }
        }
    }
    return arr
}
```

**GoDoc comments.** Every exported function and type has a documentation comment that starts with the symbol name, following the Go documentation standard. This generates correct documentation on pkg.go.dev and the repository website.

**Tests and benchmarks.** Each algorithm has a corresponding `_test.go` file. Most packages include both unit tests and benchmark functions so you can compare implementations under realistic conditions.

**No external dependencies.** The `go.mod` declares only `go 1.19` with no `require` directives. Every implementation uses the Go standard library only.
